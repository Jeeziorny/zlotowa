Settings

Konfiguracja aplikacji, AI i zarządzanie danymi

Na tej stronie

  1. 1. General
  2. 2. LLM (klasyfikacja AI)
  3. 3. Data

Settings ma trzy taby: General, LLM i Data. Każdy odpowiada za coś innego, więc lecę po kolei.

1 General

Na razie jest tu jedna opcja: checkbox "Show Rules tab in sidebar". Jak to zaznaczysz to w pasku bocznym pojawi się dodatkowa zakładka "Rules", w której możesz przeglądać i edytować reguły klasyfikacji (te same, które tworzą się automatycznie przy imporcie). Jeśli nie chcesz zaśmiecać sobie sidebara — zostaw odznaczone.

2 LLM (klasyfikacja AI)

Ta sekcja jest w pełni opcjonalna. Złotówa działa bez żadnego AI — możesz klasyfikować wydatki ręcznie albo polegać na regułach regexowych, które aplikacja tworzy automatycznie. Ale jeśli chcesz żeby nowe, nieznane wydatki klasyfikowały się same to tutaj podpinasz model językowy.

Jak to działa

Kiedy złotówa nie potrafi sklasyfikować wydatku za pomocą istniejących reguł, wysyła tytuł i kwotę wydatku do skonfigurowanego LLM-a i pyta go "hej, jaka to kategoria?". LLM odpowiada, wydatek dostaje kategorię, Ty oszczędzasz czas. Żadne inne dane finansowe nie opuszczają Twojego komputera — tylko tytuły i kwoty.

Dostępni providerzy

Do wyboru są trzy opcje:

  • OpenAI — wymaga klucza API (sk-...). Używa modelu gpt-4o-mini
  • Anthropic — wymaga klucza API (sk-...). Używa modelu claude-haiku
  • Ollama (local) — nie wymaga klucza, działa lokalnie. Używa modelu llama3. Wymaga zainstalowanego i uruchomionego Ollama na Twoim komputerze
Jeśli zależy Ci na prywatności i nie chcesz wysyłać nic nigdzie to Ollama jest opcją dla Ciebie. Wszystko działa lokalnie, żaden tytuł wydatku nie opuszcza Twojego komputera.

Konfiguracja

Wybierasz providera z dropdowna, wklejasz klucz API (albo w przypadku Ollamy — endpoint URL, domyślnie http://localhost:11434). Potem masz trzy guziki:

  • Save — waliduje połączenie i zapisuje konfigurację. Jeśli klucz jest zły albo serwer nie odpowiada to dostaniesz błąd
  • Test Connection — sprawdza czy połączenie działa, ale nic nie zapisuje. Taki dry run
  • Clear — pojawia się po zapisaniu konfiguracji. Usuwa klucz API i wyłącza klasyfikację AI

3 Data

Trzy sekcje: eksport CSV, historia uploadów i backup/restore.

Export CSV

Klikasz "Export CSV", wybierasz gdzie zapisać plik i gotowe. Dostajesz CSV ze wszystkimi wydatkami — kolumny: data, tytuł, kwota, kategoria. Przydatne jak chcesz coś policzyć w excelu (albo jak chcesz być dziadersem, o czym pisałem w sekcji Wydatki).

Upload History

Lista wszystkich bulk importów które zrobiłeś. Przy każdym jest nazwa pliku, data i liczba wydatków. Jak się okaże, że zaimportowałeś nie ten plik albo coś poszło nie tak — klikasz "Delete" i cały upload (ze wszystkimi wydatkami) znika. Aplikacja poprosi o potwierdzenie żebyś nie usunął czegoś przypadkiem.

Backup & Restore

To jest pewnie najważniejsza sekcja w Settings. Backup tworzy plik JSON z całą zawartością aplikacji: wydatki, reguły klasyfikacji i budżety. Restore wczytuje taki plik z powrotem.

Tworzenie backupu

Klikasz "Create Backup", wybierasz gdzie zapisać plik (domyślna nazwa: zlotowa-backup-DATA.json) i tyle. Polecam to robić regularnie, bo lepiej mieć backup i go nie potrzebować niż potrzebować i nie mieć.

Przywracanie z backupu

Klikasz "Restore from Backup", wskazujesz plik JSON. Zanim cokolwiek się stanie, aplikacja pokaże Ci preview: ile wydatków, reguł, kategorii i budżetów jest w backupie.

Restore nie nadpisuje istniejących danych — łączy je. Duplikaty wydatków (ten sam tytuł + kwota + data) są pomijane, reguły są aktualizowane, budżety z kolidującymi datami są skipowane. Twoje obecne dane są bezpieczne.

Mimo to, aplikacja zaproponuje Ci zrobienie backupu aktualnych danych przed przywróceniem — i polecam to zrobić. Potem zaznaczasz checkbox "I want to proceed with the restore" i klikasz "Restore". Po zakończeniu zobaczysz podsumowanie: ile wydatków dodano, ile pominięto, ile reguł i budżetów wrzucono.

Backup/restore przydaje się nie tylko do robienia kopii zapasowych. Możesz go użyć żeby przenieść dane na inny komputer — wystarczy zrobić backup na starym, wrzucić plik na nowy i zrobić restore.
